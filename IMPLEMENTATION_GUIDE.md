# PartTracker v1 - Detailní Implementační Průvodce\n\n## 📦 Co je Hotovo\n\nV repozitáři je již vše pro backend a logiku:\n\n✅ Models (4 soubory) - datové struktury\n✅ Services (5 souborů) - business logic\n✅ Project config (.csproj) - dependencies\n✅ Git config (.gitignore, README)\n\n**Stav**: Plně funkční bez UI\n\n---\n\n## 🎨 Zbývající Práce: Views (UI)\n\nNeed to create 6 WinUI 3 files:\n\n### 1. Views/MainWindow.xaml + cs\n```csharp\n// MainWindow.xaml.cs\nusing Microsoft.UI.Xaml;\nusing PartTracker.Services;\n\nnamespace PartTracker\n{\n    public sealed partial class MainWindow : Window\n    {\n        public MainWindow() => InitializeComponent();\n\n        private async void OnWindowLoaded(object sender, RoutedEventArgs e)\n        {\n            try\n            {\n                var db = new DatabaseService();\n                await db.InitializeAsync();\n                \n                var admin = await db.GetUserByUsernameAsync(\"admin\");\n                if (admin == null)\n                {\n                    var auth = new AuthService(db);\n                    await auth.CreateUserAsync(\"admin\", \"admin123\", \"Admin\");\n                }\n                \n                var loginPage = new Views.LoginPage();\n                loginPage.Activate();\n                this.Close();\n            }\n            catch (Exception ex)\n            {\n                var dialog = new ContentDialog\n                {\n                    Title = \"Error\",\n                    Content = $\"Init failed: {ex.Message}\",\n                    CloseButtonText = \"OK\"\n                };\n                await dialog.ShowAsync();\n            }\n        }\n    }\n}\n```\n\n### 2. Views/LoginPage.xaml + cs\n**Purpose**: Authenticate users (Admin or Mechanic)\n\n**Flow**:\n- User enters ID + Password\n- Click \"Login\"\n- If Admin → AdminPage\n- If Mechanic → SearchPage\n\n**Key Elements**:\n- TextBox: Username\n- PasswordBox: Password\n- Button: Login\n- TextBlock: Error messages\n- ProgressRing: Loading indicator\n\n### 3. Views/SearchPage.xaml + cs\n**Purpose**: Mechanic searches for parts and prints labels\n\n**Flow**:\n1. Mechanic types 5-digit code\n2. App searches SQLite (indexed)\n3. Shows Part details: Code, Name, Description\n4. Click \"Print\" → A6 label prints\n5. Part logged as \"removed\"\n6. Result cleared, ready for next code\n\n**Key Elements**:\n- TextBox: Code input (Enter key triggers search)\n- Button: Search\n- Button: Print (disabled until part found)\n- StackPanel: Results display\n- Fallback: Show similar codes if not found\n\n### 4. Views/AdminPage.xaml + cs\n**Purpose**: Admin imports Excel, exports results, views logs\n\n**Tabs**:\n\n#### Tab 1: Import Excel\n- Button: Select .xlsx file\n- ProgressBar: Import progress\n- TextBlock: Status (\"Imported 1,250 parts\")\n\n#### Tab 2: Export Removed\n- Button: Export to Excel\n- Shows how many parts available\n\n#### Tab 3: Print Logs\n- ListView: All print events\n- Columns: Timestamp, Code, Status (Success/Failed)\n\n#### Tab 4: Settings\n- Button: Create new user\n- Button: Logout\n\n### 5. Views/CreateUserDialog.xaml + cs\n**Purpose**: Dialog to create new mechanic accounts\n\n**Elements**:\n- TextBox: Username\n- PasswordBox: Password\n- ComboBox: Role (Mechanic/Admin)\n- Button: Create\n\n### 6. App.xaml + cs\n**Purpose**: Application entry point\n\n```csharp\n// App.xaml.cs\nusing Microsoft.UI.Xaml;\n\nnamespace PartTracker\n{\n    public partial class App : Application\n    {\n        public Window? Window { get; private set; }\n\n        public App() => this.InitializeComponent();\n\n        protected override void OnLaunched(LaunchActivatedEventArgs args)\n        {\n            Window = new MainWindow();\n            Window.Activate();\n        }\n    }\n}\n```\n\n---\n\n## 🛠️ How to Implement Locally\n\n### Step 1: Clone repo\n```bash\ngit clone
